# 月度工作报告 (2025年10月)

**项目名称**: 面向老年MCI检测的ASR-CRF框架研究
**报告周期**: 2025年10月1日 - 2025年10月24日
**核心重点**: 语音识别 (ASR) 模型微调探索

---

## 1. 执行摘要 (Executive Summary)

本月工作核心是为老年MCI数据分析项目构建 foundational 的自动语音识别 (ASR) 组件。由于MCI核心数据仍在伦理审批流程中，我们利用现有方言数据集，成功搭建并验证了一套完整的Whisper模型微调与评估框架。

我们首先尝试了`Whisper-large-v3`模型，但发现其在小数据集上存在严重的过拟合问题（词错误率WER接近100%）。随后，我们果断切换至更适合当前数据规模的`Whisper-medium`模型，并通过数据增强技术将训练样本从151条扩展至755条。

通过四组系统的对比实验，我们验证了微调框架的有效性。最佳模型配置 (大Batch) **相较于基线模型，在字符准确率上实现了67%的相对提升**。同时，我们发现“低学习率”配置在效果接近的情况下，能**节省约60%的训练时间**。

本月最关键的发现是，**当前模型性能的主要瓶瓶颈是数据量严重不足**，即使微调后13.12%的最高准确率也远未达到实用标准。因此，下一步的核心计划是引入并整合大规模老年人语音数据集 `SeniorTalk`，为模型性能的飞跃式提升奠定基础。

## 2. 本月完成工作

### 2.1. 初步基线建立与策略调整
- **`Whisper-large-v3` 初步尝试**: 在项目初期，我们尝试微调`Whisper-large-v3`模型。实验表明，由于其参数量过大，在现有小规模数据集（151条样本）上出现了严重的过拟合，导致模型完全失效 (WER ≈ 1.0)。
- **模型策略切换**: 基于上述发现，我们迅速将基础模型切换为参数量更适合的 `Whisper-medium`，为后续所有实验奠定了正确的技术方向。

### 2.2. 数据增强与预处理
- **数据集扩充**: 为缓解数据稀疏问题，我们实施了音频数据增强策略（包括改变语速、音调和注入噪声），成功将原始的151条训练样本**扩展至755条**，实现了数据量的5倍增长。
- **建立预处理流程**: 创建了自动化的数据预处理脚本 (`scripts/1_prepare_dataset.py`)，能够处理音频与文本的对齐、方言标签注入、以及训练/验证集划分。

### 2.3. 系统化对比实验框架
- **搭建自动化实验流程**: 我们创建了4套独立的实验配置文件 (`configs/exp*.yaml`)，并开发了自动化脚本 (`scripts/run_experiments.sh`) 以便进行多组超参数的对比实验。
- **完成四组对比实验**:
    1.  **实验1 (高Rank)**: 增加LoRA层可训练参数，探索模型容量。
    2.  **实验2 (低学习率)**: 采用更保守的学习率和更多训练轮次，追求稳定收敛。
    3.  **实验3 (大Batch)**: 通过梯度累积实现更大的有效批量大小。
    4.  **实验4 (激进学习率)**: 尝试用高学习率和高Rank进行快速训练。

### 2.4. 评估体系的建立与修正
- **建立综合评估脚本**: 开发了评估脚本 (`scripts/comprehensive_evaluation.py`)，能够计算字正确率和拼音准确率。
- **评估指标修正**: 在初步评估中，我们发现评分标准存在一些问题，导致结果未能完全反映模型性能。我们对评估逻辑进行了调整和修正，确保了当前结果的可靠性。

## 3. 结果与关键发现

### 3.1. 实验性能对比

我们在纯净验证集上对四个微调模型和原始Whisper-medium基线进行了评估，结果如下：

| 排名 | 模型 | 字正确率 | 相比基线提升 | 备注 |
|:---:|:---|:---:|:---:|:---|
| 🥇 | **Exp3: 大Batch** | **13.12%** | **+5.28% (相对提升67%)** | 效果最佳 |
| 🥈 | Exp1: 高Rank | 12.67% | +4.83% | 效果与Exp2接近 |
| 🥉 | Exp2: 低学习率 | 11.77% | +3.93% | **训练时间节省60%** |
| 📊 | 原始Whisper-Medium | 7.84% | 基线 | 未经微调 |
| ⚠️ | Exp4: 激进学习率 | -1.87% | -9.71% | 训练失败，性能下降 |

### 3.2. 关键发现

1.  **微调显著有效**: 所有成功的微调实验都显著超越了原始Whisper-medium基线模型，证明了在特定领域数据上进行微调的必要性和有效性。
2.  **大Batch策略最优**: 实验3（大Batch）取得了最佳性能，表明在当前数据规模下，更大的批量大小有助于模型稳定学习和泛化。
3.  **训练效率与性能的权衡**: 实验2（低学习率）在性能上与实验1（高Rank）相差无几，但训练时间大幅缩短。这为未来需要快速迭代的场景提供了一个高性价比的方案。
4.  **数据量是核心瓶颈**: 尽管相对提升显著，但13.12%的绝对准确率远低于实际应用要求。这明确地指出，**当前最根本的限制因素是训练数据量的严重不足**。（实际上我认为是我评分写的还有些问题，之前数据集都是从b站上爬下来的，b站字幕不一定是对的，可能还是只从seniortalk来做测试比较好，我看了下实际识别下的文本和参考文本很相似的，待我这两天再de下bug）
